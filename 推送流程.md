# 推送流程

本文件提供「開發端推送 GitHub」與「現場離線 Docker 部署」的標準流程。

## 1. GitHub SSH 設定

1. 產生 SSH Key
```bash
ssh-keygen -t ed25519 -C "your_email@example.com"
```

2. 啟用 agent 並加入 key
```bash
eval "$(ssh-agent -s)"
ssh-add ~/.ssh/id_ed25519
```

3. 複製公鑰並加到 GitHub
```bash
cat ~/.ssh/id_ed25519.pub
```
到 GitHub: `Settings -> SSH and GPG keys -> New SSH key`

4. 測試連線
```bash
ssh -T git@github.com
```

## 2. 專案推送（標準）

1. 確認分支
```bash
git branch --show-current
```

2. 檢查變更
```bash
git status
```

3. 提交變更
```bash
git add <files>
git commit -m "your message"
```

4. 推送分支
```bash
git push -u origin <branch>
```

## 3. 設定備份（不含辨識資料）

本專案已排除辨識資料庫與執行資料，不會上傳事件資料：
- `data/*.db`
- `storage/`
- `output/`

匯出設定：
```bash
python scripts/settings_backup.py export
```

還原設定：
```bash
python scripts/settings_backup.py import --file config/settings_backup.json
```

## 4. Docker 有網打包 / 現場離線部署

### 4.1 有網環境（打包）
```bash
docker compose build api
docker save -o traffic-api_latest.tar traffic-api:latest
python3 scripts/settings_backup.py export
```

### 4.2 現場無網環境（部署）
```bash
# 進入專案根目錄
cd /home/<board_user>/traffic-violation-detection

# 載入映像
docker load -i traffic-api_latest.tar

# 套用設定（不含辨識資料）
python3 scripts/settings_backup.py import --file config/settings_backup.json

# 啟動服務
docker compose up -d
```

### 4.3 檢查服務
```bash
docker ps
docker logs -f traffic-api
curl http://127.0.0.1:8000/api/health
python3 scripts/smoke_check.py --base-url http://127.0.0.1:8000 --timeout 60

# 一鍵重啟 + 驗證
./scripts/restart_and_verify.sh http://127.0.0.1:8000 60
```

### 4.4 建議上板流程（SOP）
1. 開發機執行 4.1，產出：
- `traffic-api_latest.tar`
- `config/settings_backup.json`

2. 透過 `scp` 或隨身碟搬到板端：
```bash
scp traffic-api_latest.tar <board_user>@<board_ip>:/home/<board_user>/deploy/
scp config/settings_backup.json <board_user>@<board_ip>:/home/<board_user>/deploy/
```

3. 板端部署：
```bash
cp /home/<board_user>/deploy/settings_backup.json ./config/settings_backup.json
cp /home/<board_user>/deploy/traffic-api_latest.tar ./traffic-api_latest.tar
docker load -i ./traffic-api_latest.tar
python3 scripts/settings_backup.py import --file ./config/settings_backup.json
./scripts/restart_and_verify.sh http://127.0.0.1:8000 60
```

4. 若異常，回滾上一版：
```bash
docker images | rg traffic-api
docker tag traffic-api:<old_tag> traffic-api:latest
./scripts/restart_and_verify.sh http://127.0.0.1:8000 60
```

## 5. 建議實務

- 開發機用 SSH 推送 GitHub。
- 現場機只做部署與 pull，不做 push。
- 每次版本發佈前，先執行一次 `settings_backup.py export`。
